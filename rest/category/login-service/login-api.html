login-api

📄️ Self register (create user) using registration token

Given user data and a registration token, this will create a new user for the registration. The user will also be created in the backend. For the creation to succeed, the registration token and the registration may not be expired yet.

📄️ Link new authentication with current user

Links a new authentication with a strategy instance with the currently logged in user. For future logins using that authentication the user will be directly found A (still) valid registration token is needed. After a successful linking, the expiration of the activeLogin and loginData will be updated accoringly

📄️ Links new authentication with any user by id

Links a new authentication with any user specified by id Needs admin permissions A (still) valid registration token is needed. After a successful linking, the expiration of the activeLogin and loginData will be updated accoringly

📄️ List all users in the system

Gets a list of all users in the system. Needs admin permissions.

📄️ Create a new user without authentication

Create a new user object (which has no authentications yet) based on the given data. Needs admin permissions. Meant for admins to create users which can then be linked a authentication by a admin. Can be used, if self registration os turned off for all strategies globally.

📄️ Get logged in user details

Gets the user object of the logged in user that is sending the request.

📄️ Get any user object by id

Get the user object of any user by its id. Needs admin permission for any user other than the one sending the request (equivalen to self query).

📄️ NOT IMPLEMENTED! Update an existing user object

**NOTE**: Not implemented yet. Will always fail. Updates an existing user object using the given data. Only the entries that are given in the input will be updated in the user. Needs Admin permissions

📄️ NOT IMPLEMENTED! Update an existing user object

**NOTE**: Not implemented yet. Will always fail. Permanently deletes an existing user by its id. Needs Admin permissions

📄️ List all loginData of one user

Gets the list of all login data of a single user specified by id. Needs admin permission for any user other than the one sending the request (equivalen to self query).

📄️ List all strategies/strategy types

Returns all known strategies (strategy types) that are registered in the system

📄️ Get one strategy (type)

Gets details for the one specified strategy type

📄️ List all strategy instances (of type)

Gets the list of all Strategy instances. If a type is specified, only the instances of that type will be retrieved. If a non existant strategy type name ist specified, no instances will be returned.

📄️ Create new strategy instance

Creates a new strategy instance. The type of the instance is set at creation and cannot change later. The type from the url takes precedene. If that is not present, the type from the body will be used. The required structure of the `instanceConfig` is specified by the strategy.

📄️ List all strategy instances (of type)

Gets the list of all Strategy instances. If a type is specified, only the instances of that type will be retrieved. If a non existant strategy type name ist specified, no instances will be returned.

📄️ Create new strategy instance

Creates a new strategy instance. The type of the instance is set at creation and cannot change later. The type from the url takes precedene. If that is not present, the type from the body will be used. The required structure of the `instanceConfig` is specified by the strategy.

📄️ Get one strategy instance

Gets one strategy instance by the specified id. If the type is specified, only instances of that type will be looked at

📄️ Edit an existing strategy instance

Updates the values of an existing strategy instance. The type can not be updated. Only values given in the body will be modified on the instance.

📄️ Delete a strategy instance

Permanently deletes the login strategy instance with the given id.

Gets one strategy instance by the specified id. If the type is specified, only instances of that type will be looked at

📄️ Edit an existing strategy instance

Updates the values of an existing strategy instance. The type can not be updated. Only values given in the body will be modified on the instance.

📄️ Delete a strategy instance

Permanently deletes the login strategy instance with the given id.

📄️ List all existing auth clients.

Gets all auth clients that exist in the system. Client secrets are not returned. Needs admin permissions

📄️ Create new auth client

Creates a new auth client. Redirect urls defaults to empty list. If no redirectURLs are specified the client will be unusable until they are updated. If `requiresSecret` is `true`, authorization as this client won't work until a client secret is added (see POST /login/client/:id/clientSecret). Needs admin persmissions.

📄️ Details of one auth client (incl. censored secrets)

Gets one specific auth clients by its uuid. This will also include the list of censored client secrets (see GET /login/client/:id/clientSecret) Needs admin permissions

📄️ Edit existing auth client

Updates the auth client with the given ID. Only parameter given in the input will be changed. If `redirectUrls` is given, it wil replace **all** previous URLs. Merging (if needed) needs to be done by the client.

📄️ Permanently delete existing auth client

Permanently deletes an existing auth client by its id.

📄️ List all client secrets (censored) of auth client

Retrieves all existing client secrets of the specified auth client. The returned secrets only contain the 5 letter prefix (censored client secret text) and the fingerprint for identification (for deleting secrets). The original secret text is **NOT** returned. It can only be retrieved once while creating the secret.

📄️ Generate and return new client secret for auth client

Generates a new client secret for the specified auth client. **NOTE**: The returned value includes the secret text that is the actual secret. It can **NOT** be retrieved later on. The censored version and fingerprint will be returned too for reference.

📄️ Delete a client secret of auch client

Deletes the client secret with the given fingerprint from the autch client with the given id
